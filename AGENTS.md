# Codex Development Guidelines

本文件是 `/Users/salty/codeProject/ai` 目录下的开发规范。在本目录及其子目录内进行开发、修改、测试、文档维护和提交时，应遵守以下规则。

## 基本工作规范

- 每次完成代码或文档修改后，应提交 git commit。
- commit message 应清晰描述本次修改内容，并在末尾追加对应 model 后缀，例如：`更新用户登录流程 gpt-5.4`。
- 当前目录是工作目录；创建新项目时，应在 `ai` 目录下创建二级目录，不要直接把新项目文件散落在当前根目录。
- 代码应达到支持生产环境使用的质量标准，保持合理的分层结构、清晰的职责边界和可维护性。
- 请遵守《阿里巴巴 Java 开发手册》相关规范。
- 不允许在代码中使用魔法值！
- 必须使用可用于生产环境的官方安全的依赖包

## 注释与日志

- 方法和类上应编写注释，注释内容至少包含：
  - 基本功能描述
  - 创建日期
  - author，统一使用 `sunshengxian`
- 代码中应添加合适的注释，解释关键业务逻辑、复杂分支、边界处理和重要设计决策。
- 代码中应添加合适的日志。
- 日志级别只允许使用 `info` 和 `error`。
- 对处理流程较长的代码，应补充进度类日志，便于线上排查和追踪执行状态。

## 文档与资源

- 大模块或重要功能发生修改时，应同步完善和补充项目根目录的 `README.md`。
- 新增功能涉及的 SQL 脚本，应统一放在项目根目录的 `resources` 文件夹下。
- 允许在 `test` 目录下编写测试用例。

## 依赖与测试约束

- 不允许在项目主体代码中编写 demo 代码或测试代码。
- 不允许使用非官方依赖包。
- 测试代码应放在 `test` 目录或项目既有测试目录结构内。
